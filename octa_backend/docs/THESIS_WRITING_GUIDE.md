```python
# U-Net 编码器示例
class UNetEncoder(nn.Module):
    def __init__(self):
        super().__init__()
        # 第一层：256x256 -> 256x256
        self.enc1 = DoubleConv(3, 64)
        # 第二层：256x256 -> 128x128
        self.enc2 = DoubleConv(64, 128)
        # ... 更多层
    
    def forward(self, x):
        # 保存跳跃连接
        skip1 = self.enc1(x)
        x = self.pool(skip1)
        # ... 继续处理
        return x, [skip1, skip2, ...]
```

## ✍️ 论文语言要求

### 学术规范

**时态选择：**
- 描述方法/系统：一般过去时或一般现在时
- 表述研究成果：一般过去时
- 阐述普遍事实：一般现在时

**例句：**
```
错误：We will implement a U-Net model...
正确：We implemented a U-Net model that...

错误：The system is designed for image segmentation.
正确：The system was designed for OCTA image segmentation.
```

### 常见表达

| 功能 | 表达方式 |
|-----|---------|
| 说明目的 | aims to..., is designed to... |
| 描述方法 | proposes..., presents..., implements... |
| 表述结果 | achieves..., reaches..., obtains... |
| 指出不足 | has limitations..., does not consider... |
| 对比差异 | compared with..., in contrast to... |

### 避免的表达

```
❌ 不规范：
- 我们的系统很好
- 这个方法效果不错
- 可能会有更好的办法

✅ 规范：
- Our system achieves a Dice coefficient of 0.89
- This method outperforms the baseline by 4.5%
- Future work can explore alternative architectures
```

### 易出现的问题

**❌ 常见问题和改进：**

1. **过度技术细节**
   ```
   ❌ "我们在第3层使用了ReLU激活函数..."
   ✅ "引入注意力机制增强关键特征的感受野"
   ```

2. **缺乏对标对比**
   ```
   ❌ "系统可以分割血管"
   ✅ "系统的Dice系数为0.89，相比基础U-Net提升4.5%"
   ```

3. **实验结果不完整**
   ```
   ❌ "测试表明方法有效"
   ✅ "在100个OCTA图像上测试，平均Dice=0.89，标准差=0.03"
   ```

4. **创新点不清晰**
   ```
   ❌ "使用深度学习进行图像分割"
   ✅ "针对OCTA小血管的特点，加入多尺度特征融合和空间注意力机制"
   ```

## 🎓 论文总体建议

1. **逻辑递进**
   - 从背景→现状→问题→方案→实现→验证→结论
   - 每章都应对前章有补充和深化

2. **重点突出**
   - 第4章（设计）和第5章（实现）是重点，篇幅应最大
   - 系统架构图要清晰，要多出现
   - 核心创新点要重复强调

3. **数据支撑**
   - 每个性能声称都要有数字
   - 对比要清晰（表格形式更好）
   - 结论要有量化依据

4. **可读性**
   - 长段落要分成多个短段
   - 重要内容用列表、表格展现
   - 每章有小结，文章有结论

5. **完整性检查**
   - 检查是否有自相矛盾的说法
   - 确保图表都被文中引用
   - 验证参考文献都在正文中出现
